Every failure mode has an automatic fallback — the clinical engine never goes dark at the bedside
Firestore unavailable
→
In-memory dict (_mem) serves all reads and writes. _pending_sync[] queues failed writes. _patch_mem reloads data on next access if process crashes. Maximum data loss window: ~30 seconds.
Gemini Flash fails (3 retries)
→
Backoff: 1s → 2s → 4s. After all retries: 22 regex fallbacks extract all 18 clinical fields (mode, TV, PEEP, FiO₂, RR, Pplat, pH, PaCO₂, PaO₂, SpO₂, BP, HR, MAP, GCS, SBT, cough, vaso, prior_paco2). Full SCCM reasoning runs. Doctor sees warning banner — not silence.
Gemini Live closes (1000/1011)
→
Auto-reconnect loop fires (invisible to browser WebSocket). Exponential backoff: 2s→4s→8s→16s→30s cap. No hard attempt limit — never gives up. pending_speech_inject[] replayed on successful reconnect.
WebSocket offline
→
REST /analyze fallback (POST with full pipeline). 10-second fallback timer activates on turn_complete. Doctor loses real-time voice but retains full SCCM reasoning and card output.
Arabic speech / accent
→
Bilingual normalization layer: Arabic phonetic medical term map (80+ terms covering ICU parameters, modes, number words) + Latin transliteration map (20+ patterns: "peap"→PEEP, "plato"→plateau, "beep"→PEEP). Runs before any extraction, regardless of model version. Produces a bilingual ICU voice agent working in Arabic and English simultaneously.
